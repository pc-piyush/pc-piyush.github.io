Reflexion: Language Agents with Verbal Reinforcement Learning

Piyush Chaudhari

Current Problem

Recent works demonstrate that LLMs can act as autonomous agents:

System Description
ReAct Reason + Act in text-based environments
SayCan Grounding language in robotic affordances
Toolformer LLMs that teach themselves to use API tools
WebGPT Browser-assisted Q&A with human feedback
HuggingGPT Coordinating AI models via ChatGPT
Generative Agents Simulating believable human behaviour

All of these rely on in-context few-shot examples as the only form of teaching — because traditional RL requires weight updates the agents cannot do at inference time.

Issue with Traditional Reinforcement Learning

Traditional RL requires:

  • Millions of environment interactions
  • Expensive gradient-based fine-tuning
  • Massive compute budgets
  • Retraining for each new task

What LLM agents have today:

  • In-context few-shot examples only
  • No weight updates at inference
  • No persistent learning across episodes
  • Frozen model parameters

The Gap: LLM agents can observe that they failed — but they cannot update their parameters. Without a mechanism to convert failure signals into actionable learning, every episode starts from scratch.

Approach

Reflexion draws inspiration from human metacognitive learning:

\[ ATTEMPT → FAIL → REFLECT → STORE → RETRY \]

  1. ATTEMPT — Try the task with current knowledge
  2. FAIL — Receive feedback (binary, scalar, or free-form)
  3. REFLECT — Identify why the attempt failed (credit assignment)
  4. STORE — Record key insight in long-term memory
  5. RETRY — Apply lessons in the next attempt

Reflexion Formalises This Loop: Instead of updating neural weights, Reflexion converts feedback into verbal summaries stored in an episodic memory buffer — acting as a semantic gradient that guides the next attempt.

Reflexion : Core Concept

Reflexion reinforces language agents not by updating weights, but by converting feedback into verbal language stored in memory, used to guide better decisions in future trials.

Six key advantages:

Property Detail
Lightweight No fine-tuning or gradient descent — works with any frozen LLM
Nuanced Feedback Verbal explanations carry more information than scalar rewards
Interpretable Self-reflections can be inspected, audited, or monitored
Flexible Works with binary, scalar, or free-form feedback from any source
Few-shot Learning Strong improvements within just a handful of trials
Language Agnostic Demonstrated on Python AND Rust code generation tasks

Redefining the Policy

Traditional RL policy:

\[\pi_\theta(a_t \mid s_t), \quad \theta = \text{model weights}\]

Reflexion policy:

\[\pi_\theta(a_t \mid s_t), \quad \theta = \{M_a,\ \textit{mem}\}\]

Where:

  • \(M_a\) = the Actor LLM (frozen weights)
  • \(\textit{mem}\) = episodic memory buffer of self-reflections

Key Insight: By adding mem as a second policy parameter, Reflexion “optimises” agent behaviour through verbal learning — without ever touching LLM weights. The self-reflection text acts as a semantic gradient signal.

Reflexion: Three-Model Architecture

Model Symbol Role
Actor \(M_a\) Generates text and actions from state observations
Evaluator \(M_e\) Scores the Actor’s trajectory; produces reward \(r_t\)
Self-Reflection \(M_{sr}\) Converts sparse reward into verbal feedback \(sr_t\)

Memory : Short-Term vs Long-Term

Short-Term Memory (trajectory):

  • Current trial’s trajectory history
  • Contains: \(\tau_t = [a_0, o_0, a_1, o_1, \ldots, a_n, o_n]\)
  • Actions \(a_i\) and observations \(o_i\) from the ongoing episode
  • Resets at the start of each new trial
  • Kept in the LLM’s active context window

Long-Term Memory (mem):

  • Accumulated self-reflection outputs \(\{sr_0, sr_1, \ldots\}\)
  • Persists across trials as a sliding window
  • Bounded by \(\Omega = 1\text{–}3\) most recent reflections (to fit context)
  • Provides “self-hints” for the next episode
  • Key advantage over other LLM agent approaches

Memory is capped at \(\Omega\) experiences to avoid exceeding LLM context window limits.

The Reflexion Algorithm

Short-Term Memory (trajectory):

  • Initialise Actor, Evaluator, Self-Reflection: \(M_a, M_e, M_sr\)

  • Initialise policy \(π_θ(a_t | s_t), θ = {M_a, mem}\)

  • Generate initial trajectory \(τ₀\) using \(π_θ\)

  • Evaluate \(τ₀\): \(r₀ = M_e(τ₀)\)

  • Self-reflect: \(sr₀ = M_sr(τ₀, r₀)\)

  • Set mem ← \([sr₀]\)

  • Set t = 0

──────────────────────────────────────────────

while \(M_e(τ)\) ≠ pass OR \(t\) < max_trials:

  • Generate \(τ_t\) = \([a₀, o₀, … a_i, o_i]\) using \(π_θ\)

  • Evaluate: \(r_t = M_e(τ_t)\)

  • Self-reflect: \(sr_t = M_sr(τ_t, r_t, mem)\)

  • Append \(sr_t\) to mem

  • Increment \(t\)

──────────────────────────────────────────────

return

Experiments

Decision-Making — AlfWorld Setup

Environment: AlfWorld — text-based household tasks built on TextWorld.

134 environments across 6 task types:

  • Find hidden objects (e.g., spatula in a drawer)
  • Move objects to target locations
  • Manipulate objects with other objects
  • Examine items under a light source

Agent configuration:

  • Actor: ReAct (reasoning + action generation)
  • Evaluator: LLM-based binary classification OR hand-written heuristic
  • Heuristic rule: same action repeated ≥ 3 times, OR > 30 steps taken → trigger reflection
  • Memory: last \(\Omega = 3\) self-reflections

Two failure modes Reflexion learns to fix:

  1. Hallucination — agent thinks it has an item it doesn’t possess
  2. Inefficient planning — agent circles through same locations repeatedly

Unit test generation pipeline:

1. CoT prompting → diverse test suite with natural language descriptions
2. Filter: keep only tests with valid Abstract Syntax Tree (AST)
3. Sample n ≤ 6 tests → T = {t₀, t₁, …, tₙ}
4. Execute against generated code → pass/fail → feeds self-reflection

Unit Test provide grounded, execution-verified feedback — stronger than LLM self-evaluation alone, and eligible for pass@1 accuracy reporting.

Worked Example — AlfWorld Trial & Reflection

Task: Examine the mug with the desklamp.

Trial #1 — FAIL:

Think: find mug first, then find desklamp
→ go to drawer 1..6 (mug not found)
→ go to desk 1: find mug 1, take it ✓
→ go to desk 2 looking for desklamp
→ use desklamp 1: Nothing happens ✗
→ use desklamp 1: Nothing happens ✗
 
Status: FAIL

Self-Reflection generated by \(M_{sr}\):

“I tried to find the mug before the desklamp. I should have looked for the desklamp first — I noticed it was on desk 1. Next trial: go to desk 1, pick up the lamp, then find and examine the mug with it.”

Trial #2 — SUCCESS:

[mem loaded: look for desklamp first]
→ go to desk 1
→ sees desklamp 1 AND mug 1 (both here!)
→ take mug 1 from desk 1
→ use desklamp 1: "You turn on the desklamp" ✓
 
Status: SUCCESS  (3 actions vs 10+ failed attempt)

Ablation Studies

Code Generation — What Drives the Gains? (HumanEval Rust, 50 hardest)

Variant pass@1 Description
Base model 0.60 Single generation, no Reflexion
No test generation 0.52 Reflection only, no unit tests
No self-reflection 0.60 Tests only, no verbal explanation
Full Reflexion 0.68 Tests + self-reflection

Analysis of each omission:

  • No test generation → 0.52 (worse than baseline)
    Without unit tests the agent cannot assess whether its code is correct. It must participate in all iterations — even successful ones — causing harmful edits to working code.

  • No self-reflection → 0.60 (same as baseline)
    Tests catch syntax and logic errors, but without the verbal explanation step, the agent cannot translate the error signal into an implementation fix. The two components are synergistic.

  • Full Reflexion → 0.68
    Tests answer what is wrong. Self-reflection synthesises why and proposes a fix.

Tests alone ≠ improvement. Reflection alone < baseline. Both together = new SOTA.

Ablation Studies

Where Reflexion Breaks Down — WebShop

Why does Reflexion fail here?

Reason Explanation
Local minima Reflexion optimises from its current trajectory; cannot escape a bad search strategy
Needs diversity WebShop requires very creative, unique search queries — Reflexion produces repetitive reflections
Ambiguous space E-commerce NL search is highly ambiguous; Reflexion cannot bridge failed query semantics
Contrast: AlfWorld AlfWorld succeeds because permissible actions are enumerable in observations; WebShop is unbounded

Key limitation: Reflexion is a local optimiser. Tasks that require radical exploration or very high action space diversity are out of its reach.

Limitations

Limitation Detail
Local minima Verbal RL converges to locally-optimal solutions; no global optimality guarantee
LLM quality ceiling Self-correction is an emergent property of strong models only — starchat-beta shows zero improvement
Context window cap Memory bounded by \(\Omega\); future work: vector DB or SQL-backed external memory
Flaky test suite Programming tasks depend on self-generated tests; flaky tests mislead the agent
No formal convergence Unlike classical RL, no proof that Reflexion terminates within finite trials
Safety / misuse risk Greater autonomy = greater risk; self-reflections need monitoring before tool use

Starchat-beta ablation (shows LLM quality dependency):

Approach pass@1 (avg over 8 trials) pass@1 (std)
Baseline 0.26 0.00481
Reflexion 0.26 0.00305

Zero improvement — self-reflection is an emergent quality of stronger, larger models.

Impact and Future Directions

Interpretability advantage:

  • Unlike black-box RL, Reflexion’s self-reflections are human-readable
  • Monitoring text reflections before executing tools → alignment potential
  • Conversation logs become an auditable learning trail
  • Misuse risk: : Greater agent capability amplifies misuse risk
  • Autonomous code execution must run in isolated sandboxes
  • Safety & ethics research must co-evolve with capabilities

Future directions:

Direction Description
Vector / SQL memory Replace sliding window with scalable external memory for unbounded learning
Off-policy exploration Apply ε-greedy or UCB techniques during verbal RL trial selection
Value learning in language Learn NL value functions to estimate expected future success per stage
Formal verification Use theorem provers / formal specs to ground code evaluation beyond unit tests

Key Takeaways

  1. Verbal Reinforcement is Real — LLMs can learn within a few trials using natural language alone; no gradient descent required.

  2. Memory is the Key Ingredient — Self-reflections stored in episodic memory provide +8% accuracy over raw trajectory replay alone.

  3. Tests + Reflection are Synergistic — For code generation, neither unit tests alone nor reflection alone achieves gains; both are required.

  4. New SOTA on Code Benchmarks — 91% HumanEval Python pass@1, surpassing GPT-4 (80.1%); doubles GPT-4’s Leetcode Hard rate to 15%.

  5. Still Has Limits — Local minima, LLM capability ceiling, context window size, and flaky tests all constrain Reflexion’s applicability.
